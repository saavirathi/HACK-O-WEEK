About the Project

This system stores books in a fixed-size array, demonstrating fundamental data structure concepts. It supports adding books (only within capacity limits), removing books by title, and searching using efficient algorithms. The catalogue maintains alphabetical order for optimised search performance, all without dynamic memory allocation.

Core Capabilities

Features

Add Book

Insert a new book only if the array has available space within the fixed capacity.

Remove Book

Delete a book by title and shift remaining elements to fill the gap.

Search Book

Find books by title using linear or binary search on the sorted array.

Display All

List every book in alphabetical order with index positions shown.

Step by Step

How It Works

1

Books Stored in Array

A fixed-size array (e.g., char books[MAX][50]) holds all book titles. Capacity is set at compile time.

2

Sorted Alphabetically

After every insertion, the array is sorted using a simple sorting algorithm (e.g., insertion sort) to keep titles in order.

3

Search by Title

Binary search runs in O(log n) on the sorted array. A linear search fallback runs in O(n).

4

Fixed Capacity

No dynamic resizing — if the array is full, the system rejects new entries. This teaches memory-aware programming.
